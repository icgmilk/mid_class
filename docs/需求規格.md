# 需求規格

## 使用案例: 退選 

D1123985 許鈞翔

### 前置條件

* 學生不會輸入無效的學號
* 學生選擇退選

### 基本流程

1. 將學號輸入選課系統
2. 執行使用案例:登入系統
3. 系統提示學生選擇功能
4. 學生選擇退選
5. 學生輸入課程代碼
6. 系統檢查該課程代碼是否為空
7. 系統檢查該課程代碼是否在該學生課表內
8. 系統檢查學分數是否超出限制
9. 使用案例結束，退選課程成功

### 替代流程

6.1 於基本流程第六步，假如學生未輸入課程代碼

1. 系統提示學生課程代碼不能為空，請使用者重新輸入
2. 系統回到輸入課程代碼頁面
3. 學生重新輸入課程代碼
3. 重複執行基本流程第五步

7.1 於基本流程第七步，假如學生輸入課程代碼不在學生課表內

1. 系統提示學生輸入課程代碼不在該學生課表內
2. 系統回到選擇介面
3. 學生重新選擇功能
4. 重複執行基本流程第三步

8.1 於基本流程第八步，假如學生退選後學分數 < 最低學分數

1. 系統提示已達最低學分數
2. 系統回到選擇介面
3. 學生重新選擇功能
4. 重複執行基本流程第三步

### 後置條件

* 若使用案例結束，退選課程成功。則學生課表內的課程會減去其退選的課程；退選後學分數=退選前學分數-退選課程學分數
* 若使用案例結束，退選課程失敗。則學生課表內的課程不變；學生退選前後的學分數保持不變。

### #1

使用案例: 退選課程

系統初始狀態: 系統讓學生輸入課程代碼

測試動作: 學生沒輸入課程代碼就進入

預期結果: 

1. 系統提示學生未輸入課程代碼
2. 系統回到選擇功能的介面

### #2

使用案例: 退選課程

測試功能: 系統會檢查學生輸入的課程代碼是否在課表內

系統初始狀態: 系統讓學生輸入課程代碼

測試動作: 學生輸入不在課表內的課程代碼

預期結果: 

1. 系統提示學生輸入的課程代碼不在課表內
2. 系統回到選擇功能的介面

### #3

使用案例: 退選課程

測試功能: 系統會檢查學生退選後是否會小於最低學分數

系統初始狀態: 系統讓學生輸入課程代碼，且學生學分數-預期退選課程學分數<最低學分數

測試動作: 學生輸入課表內的課程代碼

預期結果: 

1. 系統提示學生退選此課程學分數將會小於最低學分數
2. 系統回到選擇功能的介面

# 使用案例: 加選

D1110176 吳柏宏

### 前置條件

* 學生不會輸入無效的學號
* 學生選擇加選

### 基本流程

1. 將學號輸入選課系統
2. 執行使用案例:登入系統
3. 系統提示學生選擇功能
4. 學生選擇加選
5. 學生輸入課程代碼
6. 系統檢查課程代碼是否為空
7. 系統檢查課程是否存在
8. 系統檢查課程是否已加選
9. 系統檢查學分數是否超出限制
10. 系統檢查課程是否衝堂
11. 系統提示加選成功
12. 使用案例結束，加選課程成功

### 替代流程

6.1 於基本流程第六步，假如學生未輸入課程代碼

1. 系統提示學生未輸入課程代碼，請使用者重新輸入
2. 回到基本流程第五步

7.1 於基本流程第七步，假如學生輸入課程代碼不存在對應課程

1. 系統提示學生找不到對應課程
2. 回到基本流程第三步

8.1 於基本流程第八步，假如學生已經加選過該課程

1. 系統提示該課程已選
2. 回到基本流程第三步

9.1 於基本流程第九步，假如學生已選學分數超出限制

1. 系統提示學生學分已滿
2. 回到基本流程第三步

10.1 於基本流程第十步，假如學生欲選課程與已選課程衝堂

1. 系統提示學生課程衝堂
2. 回到基本流程第三步

### 後置條件

* 若使用案例結束，加選課程成功。則學生課表內的課程會加入其加選的課程；加選後學分數=加選前學分數+加選課程學分數
* 若使用案例結束，加選課程失敗。則學生課表內的課程不變；學生加選前後的學分數保持不變。

測試動作: 學生沒輸入課程代碼就進入

預期結果: 

1. 系統提示學生未輸入課程代碼
2. 系統回到選擇功能的介面

## 使用案例: 登入系統

### 前置條件

* 學生具有唯一的個人學號

### 基本流程

1. 將學號輸入選課系統
2. 系統驗證學生學號是否存在
3. 系統登入成功

### 替代流程

1.1 於基本流程第一步，假如檢驗學生未輸入學號

1. 系統提示學生未輸入學號，請學生輸入學號
2. 學生重新輸入學號
3. 重複進行基本流程第一步

2.2 於基本流程第二步，假如檢驗學生學號不存在

1. 系統提示學生輸入之學號不存在
2. 系統停止執行
